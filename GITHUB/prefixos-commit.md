# **Padrão Conventional Commits (mais comuns)**

- feat: Nova funcionalidade
- fix: Correção de bug
- docs: Alteração em documentação
- style: Formatação (espaço, ponto e vírgula, lint)
- refactor: Refatoração sem mudar comportamento
- test: Testes
- chore: Manutenção (deps, configs, scripts)
- ci: CI/CD (pipelines, workflows)
- build: Build, Docker, dependências
